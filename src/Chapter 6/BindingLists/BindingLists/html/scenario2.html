This example builds a WinJS.Binding.List on top of an array of objects (each of which contains a few random numbers), optionally using the binding and proxy constructor options. The raw array as well as the List are shown in the output.

The two upper buttons change the corresponding collection by replacing objects. The relationships between the array and List are the same as in scenario 1, affected again by the constructor options (changing either checkbox will recreate the List from the current array):

  • Binding will make each object in the list observable. This is demonstrated by lower control that's data-bound to the List's first item.
  • Proxy will, as in scenario 1, update the array whenever the List is modified.

The lower two buttons change the corresponding collection by changing object properties. In both cases, changes are reflected both ways even without Proxy checked, as both collections maintain references to the same in-memory object.
